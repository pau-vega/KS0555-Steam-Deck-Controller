# Research: Switch release-please to PAT

## Source
[googleapis/release-please-action README](https://github.com/googleapis/release-please-action)

## Key Findings

### 1. PAT is the recommended default
The official README uses PAT in every example:
```yaml
token: ${{ secrets.MY_RELEASE_PLEASE_TOKEN }}
```
Comment: "this assumes that you have created a personal access token (PAT) and configured it as a GitHub action secret."

### 2. GITHUB_TOKEN limitation is confirmed in docs
> "When you use the repository's GITHUB_TOKEN to perform tasks, events triggered by the GITHUB_TOKEN will not create a new workflow run. This prevents you from accidentally creating recursive workflow runs."
> "You will want to configure a GitHub Actions secret with a Personal Access Token if you want GitHub Actions CI checks to run on Release Please PRs."

### 3. Secret name is arbitrary
Docs use `MY_RELEASE_PLEASE_TOKEN` as example. Any name works. Use `RELEASE_PLEASE_PAT` for clarity.

### 4. `pr` output is a JSON string
`pr` output = "A JSON string of the PullRequest object" — confirms `fromJSON` fix is correct.

### 5. No other workflow changes needed
Only the `token` value changes. Permissions, triggers, and job logic stay the same.

## Action Items
1. User creates a classic PAT with `repo` scope, saves as `RELEASE_PLEASE_PAT` secret
2. Change `release-please.yml` line 31: `token: ${{ secrets.GITHUB_TOKEN }}` → `token: ${{ secrets.RELEASE_PLEASE_PAT }}`
3. Push + PR
