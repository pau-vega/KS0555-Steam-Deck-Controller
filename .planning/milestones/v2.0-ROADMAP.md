# Milestone v2.0: Tauri Migration

**Status:** ✅ SHIPPED 2026-05-12
**Phases:** 6-10
**Total Plans:** 12

## Overview

Migrate apps/frontend from browser-based React+Vite to a Tauri v2 desktop app, replacing broken Web Bluetooth and Gamepad APIs with native Rust alternatives (btleplug + gilrs). All frontend hooks preserve stable interfaces so app.tsx, control-pad.tsx, and status-bar.tsx remain unchanged.

### Phase 7: BLE Commands with btleplug

**Goal**: Rust BLE module implemented with Tauri commands for BT24 robot communication
**Depends on**: Phase 6
**Plans**: 3 plans

Plans:

- [x] 07-01-PLAN.md — BLE Connection Command with State Management (BLE-01, BLE-04, BLE-05)

- [x] 07-03-PLAN.md — Tauri Permissions and Linux Filtering (BLE-06)

**Details:**
- BleState struct with Arc<Mutex<Option<Peripheral>>> for thread-safe peripheral storage
- ble_connect command: 5s scan timeout, post-filter by "BT24" name, emits "ble-state-changed"
- ble_send: writes to characteristic 0000ffe1 using WriteType::WithoutResponse
- ble_disconnect: disconnects and emits "ble-state-changed" with "disconnected"
- CentralEvent::DeviceDisconnected auto-emits "ble-state-changed" with "disconnected"
- Linux BlueZ filter merged across clients — post-filter by device name

## Milestone Summary

**Key Decisions:**

- Decision: Tauri v2 over v1 (Rationale: v2 is current stable, better SteamOS support)
- Decision: btleplug for BLE (Rationale: Cross-platform Rust BLE crate, works on Linux/SteamOS)
- Decision: gilrs for gamepad (Rationale: Rust gamepad library, sees Steam Deck built-in controller)
- Decision: Replace Web Bluetooth API (Rationale: WebKitGTK on SteamOS blocks navigator.bluetooth)
- Decision: Replace Gamepad API (Rationale: Steam Input intercepts before WebView gets it)
- Decision: Keep hook return shapes stable (Rationale: app.tsx/control-pad.tsx/status-bar.tsx must be unchanged)
- Decision: Monorepo preserved (Rationale: src-tauri lives inside apps/frontend/)
- Decision: Deprecate apps/backend (Rationale: Fastify + WebSocket no longer needed)

**Issues Resolved:**

- Web Bluetooth API does not work in Tauri's WebKitGTK on Linux/SteamOS — replaced with btleplug
- Steam Input intercepts built-in gamepad before WebView via navigator.getGamepads() — replaced with gilrs
- BlueZ discovery filters merge across all D-Bus clients on Linux — post-filter by device name

**Technical Debt Incurred:**

- Custom tauri-cli fork used for AppImage (resolved in v2.1 by switching to deb)
- Phase 10 CI only builds AppImage (resolved in v2.1 by Flatpak pipeline)
