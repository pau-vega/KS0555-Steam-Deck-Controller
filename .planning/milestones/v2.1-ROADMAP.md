# Milestone v2.1: Flatpak Packaging

**Status:** ✅ SHIPPED 2026-05-12
**Phases:** 11-19
**Total Plans:** 15

## Overview

Replace AppImage distribution with a sideloaded Flatpak bundle for Steam Deck. BLE + gamepad work inside the Flatpak sandbox; "Add as Non-Steam Game" launches in Gaming Mode without regression. CI produces deb → Flatpak artifact chain end-to-end.

## Milestone Summary

**Key Decisions:**

- Flatpak runtime choice: org.freedesktop.Platform//24.08 with SDK 24.08 (PKG-04)
- Sideload-only distribution (no Flathub submission)
- Auto-update reframed as manual upgrade workflow + optional polling script
- in_flatpak() detection via FLATPAK_ID + /.flatpak-info (belt-and-suspenders)
- D-Bus rewrite gated behind !in_flatpak()
- WEBKIT_DISABLE_COMPOSITING_MODE remains unconditional
- Anti-feature checklist as comment block in manifest
- upgrade-robot-controller.sh at repo root (curl + jq, zero deps)
- Single-job CI (no more build-x64/build-flatpak-x64 split)
- Version from Cargo.toml via cargo metadata + jq
- VAL-08 git diff lock dropped from CI (pre-commit hooks enforce), later not in lefthook.yml (deferred)
- Stale Tauri cache cleanup before cargo tauri build

**Issues Resolved:**

- PKG-03: cargo tauri build --bundles deb executed successfully (3.6 MB .deb)
- VAL-05: flatpak-builder run successfully in CI (2.4 MB .flatpak)
- Runtime mismatch (GNOME 48 → Freedesktop 24.08) fixed
- Stale Tauri cache absolute-path drift fixed with cleanup step
- Stale docs (STEAM_DECK.md, ARCHITECTURE.md) rewritten for Flatpak era
- All phases 13, 15, 16 now have VERIFICATION.md

**Issues Deferred:**
