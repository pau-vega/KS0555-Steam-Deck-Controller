    1. `mkdir -p` — create all target directories under `/app/`
    2. `ar -x` — extract deb archive (produces `debian-binary`, `control.tar.*`, `data.tar.*`)
    3. `tar -xf data.tar.*` — extract the actual app payload from the data tarball (glob handles any compression)
    4. `install -Dm755 usr/bin/robot-controller` → `/app/bin/robot-controller` — binary, executable
    5. `install -Dm644 usr/share/applications/robot-controller.desktop` → `/app/share/applications/com.ks0555.robotcontroller.desktop` — desktop file, RENAMED to Flatpak ID
    6. `sed -i 's/^Icon=.*/Icon=com.ks0555.robotcontroller/'` — update Icon= line in renamed desktop file to match the Flatpak ID (Tauri writes `Icon=robot-controller` but Flatpak installs icons as `com.ks0555.robotcontroller.png`)
    7-9. `install -Dm644` each icon from sources to `/app/share/icons/hicolor/{size}/apps/com.ks0555.robotcontroller.png`
    10. `install -Dm644 com.ks0555.robotcontroller.metainfo.xml` → `/app/share/metainfo/`

    **`install -Dm644` vs `install -Dm755`:**
    - `755` for executables (binary)
    - `644` for data files (desktop, icons, metainfo)

    Create `flatpak/build.sh` — executable shell script that builds the Flatpak bundle.

    **Script behavior:**

    ```bash
    #!/usr/bin/env bash
    # build.sh — Build the Robot Controller Flatpak bundle
    # Usage: build.sh <path-to-deb>
    #   <path-to-deb> — absolute or relative path to the .deb from Tauri build
    #   e.g.: build.sh apps/frontend/src-tauri/target/release/bundle/deb/robot-controller_0.1.5_amd64.deb
    set -euo pipefail

    # === Argument validation ===
    if [ $# -ne 1 ]; then
        echo "Usage: build.sh <path-to-deb>"
        echo "  <path-to-deb> — path to the .deb produced by 'cargo tauri build --bundles deb'"
        exit 1
    fi

    DEB_PATH="$1"

    if [ ! -f "$DEB_PATH" ]; then
        echo "Error: deb file not found at '$DEB_PATH'"
        echo "Run 'cargo tauri build --bundles deb' from apps/frontend/src-tauri/ first."
        exit 1
    fi

    # === Resolve to script directory ===
    SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
    MANIFEST="${SCRIPT_DIR}/com.ks0555.robotcontroller.yaml"
    BUILD_DIR="${SCRIPT_DIR}/build-dir"
    REPO_DIR="${SCRIPT_DIR}/repo"
    DEB_COPY="${SCRIPT_DIR}/robot-controller.deb"

    # === Copy deb into flatpak/ directory ===
    # Manifest references 'robot-controller.deb' as a relative path source,
    # so the deb must be in the same directory as the manifest during build.
    cp "$DEB_PATH" "$DEB_COPY"
    echo "→ Copied deb to $DEB_COPY"

    # === Platform detection ===
    case "$(uname -s)" in
        Darwin)
            echo "→ macOS detected — running structural validation only (D-16)"
            perform_structural_validation
            ;;
        Linux)
            echo "→ Linux detected — running flatpak-builder"
            perform_flatpak_build
            ;;
        *)
            echo "Error: unsupported platform '$(uname -s)'"
            exit 1
            ;;
    esac
    ```

    **`perform_structural_validation()` function (macOS path):**
    ```bash
    perform_structural_validation() {
        echo "→ Checking manifest YAML syntax..."
        if command -v python3 &>/dev/null; then
            python3 -c "import yaml; yaml.safe_load(open('${MANIFEST}'))" \
                && echo "  ✓ Manifest YAML is valid" \
                || { echo "  ✗ Manifest YAML parse error"; exit 1; }
        else
            echo "  ⚠ python3 not found — skipping YAML validation"
        fi

        echo "→ Checking metainfo XML well-formedness..."
        if command -v xmllint &>/dev/null; then
            xmllint --noout "${SCRIPT_DIR}/com.ks0555.robotcontroller.metainfo.xml" \
                && echo "  ✓ Metainfo XML is well-formed" \
                || { echo "  ✗ Metainfo XML parse error"; exit 1; }
        else
            echo "  ⚠ xmllint not found — skipping XML validation"
        fi

        echo "→ Checking icon files exist..."
        for icon in \
            "${SCRIPT_DIR}/icons/32x32/com.ks0555.robotcontroller.png" \
            "${SCRIPT_DIR}/icons/128x128/com.ks0555.robotcontroller.png" \
            "${SCRIPT_DIR}/icons/256x256@2/com.ks0555.robotcontroller.png"; do
            if [ -f "$icon" ]; then
                echo "  ✓ $(basename "$(dirname "$icon")")/$(basename "$icon") exists"
            else
                echo "  ✗ Missing: $icon"
                exit 1
            fi
        done

        echo "→ Checking metainfo file exists..."
        if [ -f "${SCRIPT_DIR}/com.ks0555.robotcontroller.metainfo.xml" ]; then
            echo "  ✓ com.ks0555.robotcontroller.metainfo.xml exists"
        else
            echo "  ✗ Missing metainfo.xml"
            exit 1
        fi

        echo ""
        echo "✓ macOS structural validation passed"
        echo "  To build the Flatpak, run this script on a Linux machine"
        echo "  or on the Steam Deck with flatpak-builder installed."
    }
    ```

    **`perform_flatpak_build()` function (Linux path):**
    ```bash
    perform_flatpak_build() {
        echo "→ Cleaning previous build artifacts..."
        rm -rf "$BUILD_DIR" "$REPO_DIR"

        echo "→ Running flatpak-builder..."
        flatpak-builder \
            --user \
            --install \
            --force-clean \
            "$BUILD_DIR" \
            "$MANIFEST"

        echo "→ Creating flatpak-builder repo..."
        mkdir -p "$REPO_DIR"
        flatpak build-export "$REPO_DIR" "$BUILD_DIR"

        echo "→ Building single-file .flatpak bundle..."
        flatpak build-bundle \
            "$REPO_DIR" \
            "${SCRIPT_DIR}/../RobotController-x86_64.flatpak" \
            com.ks0555.robotcontroller

        echo ""
        echo "✓ Flatpak build complete"
        echo "  Bundle: RobotController-x86_64.flatpak"
        echo "  Install: flatpak install --user RobotController-x86_64.flatpak"
        echo "  Run:     flatpak run com.ks0555.robotcontroller"
    }
    ```

    **After writing, make executable:**
    ```bash
    chmod +x flatpak/build.sh
    ```

    **Key design decisions (traceable):**
    - D-15: Script takes exactly one argument `<path-to-deb>`. No auto-location, no default path.
    - D-16: macOS = structural validation (YAML + XML + file existence). Linux = full flatpak-builder. Unsupported platform = error exit.
    - D-02: Script copies the real deb to `flatpak/robot-controller.deb` (relative to manifest) before invoking flatpak-builder. The manifest's `path: robot-controller.deb` is relative to the manifest file.
    - D-08: No pre-build manifest validation on Linux — flatpak-builder validates the manifest itself during build.
    - The `.flatpak` output goes to repo root as `RobotController-x86_64.flatpak` (relative path `../RobotController-x86_64.flatpak` from `flatpak/`).

    **Error handling:**
    - `set -euo pipefail` — exit on error, undefined variable, pipe failure
    - Missing deb → descriptive error with hint to build first
    - Unsupported platform → descriptive error
    - flatpak-builder failure → `set -e` causes script to exit with flatpak-builder's exit code
    - Cleanup on success only: build-dir and repo are cleaned at the START of the next build, not the end (keeps artifacts for inspection)

    **No cleanup at script end** — build-dir and repo persist for debugging. They're cleaned at the START of the next `build.sh` invocation.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build.sh input → flatpak-builder | The `.deb` path provided by the user is untrusted. build.sh validates the file exists but does not verify its contents — flatpak-builder and the manifest's explicit build-commands limit what gets installed. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Spoofing | deb extraction in build-commands | mitigate | Manifest `build-commands` only install files from known paths (`usr/bin/robot-controller`, `usr/share/applications/robot-controller.desktop`, `usr/share/icons/hicolor/256x256/apps/robot-controller.png`). Files at unexpected paths within the deb are not copied. No wildcard `cp -r usr/ /app/`. |
| T-12-02 | Tampering | deb file substitution between build and flatpak-builder | accept | Local development scenario. The user invokes build.sh with an explicit deb path. No network transfer. CI (Phase 15) adds SHA256 verification. |
| T-12-03 | Information Disclosure | build.sh logging | accept | build.sh prints file paths and build status. No secrets, no environment variables echoed. |
| T-12-04 | Denial of Service | Unbounded build artifacts (build-dir, repo) | accept | `build-dir` and `repo` are cleaned at the start of each build. Disk usage is bounded by a single build. No accumulation across runs. |
| T-12-05 | Elevation of Privilege | flatpak-builder invocation | accept | flatpak-builder runs as the current user (not root). The `--user` flag installs to user scope only. No system-wide effects. |
</threat_model>
