<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer machine → flatpak-builder | The `.deb` input is an untrusted artifact from the Tauri build. Build-commands operate on its contents. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Spoofing | flatpak-builder build-commands | mitigate | Manifest build-commands only install files from known paths (`usr/bin/robot-controller`, `usr/share/applications/robot-controller.desktop`, `usr/share/icons/hicolor/256x256/apps/robot-controller.png`). If the deb contains files at unexpected paths, they are silently ignored (not installed). No wildcard copying of the full deb contents. |
| T-12-02 | Tampering | deb file between build and flatpak-builder | accept | Local development scenario — the user controls the deb file. No network transfer of the deb during Phase 12. CI pipeline (Phase 15) adds SHA256 verification. |
| T-12-03 | Information Disclosure | metainfo.xml, icons, README.md | accept | All assets are public metadata and icons. No secrets, credentials, or PII are embedded. |
| T-12-04 | Elevation of Privilege | build-commands during flatpak-builder | accept | flatpak-builder runs build-commands as the current user (not root) and the resulting Flatpak is sandboxed. No privilege escalation path exists from packaging artifacts alone. |
</threat_model>
